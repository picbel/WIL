# WIL Refine Skill

이 스킬은 사용자가 직접 고찰하고 작성한 WIL 원문을 전문가 수준으로 다듬고 검증합니다. **사용자의 학습 주도권을 보호하기 위해 원문이 없는 상태에서의 자동 생성은 엄격히 금지됩니다.**

## Core Command

### `/refine [file_path_or_period]`
- **목적**: 사용자가 작성한 원문의 기술적 정확성을 검토하고 문장을 전문적으로 정제합니다.
- **인자 처리 및 파일 추측 (Smart Selection)**:
  1. **연도/월 패턴 (YY/MM)**: 예) `25/03` 입력 시 `2025/03/wil.md`로 자동 변환하여 파일을 찾습니다.
  2. **전체 연도/월 패턴 (YYYY/MM)**: 예) `2024/12` 입력 시 `2024/12/wil.md`로 변환합니다.
  3. **직접 경로**: 파일 경로 전체를 입력한 경우 그대로 사용합니다.

- **제약 조건 (Critical)**:
  1. **원문 존재 확인**: 작업을 시작하기 전, 해당 파일에 사용자가 직접 작성한 내용이 있는지 확인합니다. 내용이 비어있거나 AI에게 생성을 요청하는 식의 요청은 거절합니다.
- **동작**:
  1. **기술 검증**: 작성된 개념(예: IoC, DI)이 기술 표준에 맞는지 확인하고, 오류가 있다면 수정 제안을 포함합니다.
  2. **문장 정제**: 구어체나 파편화된 문장을 기술 블로그 수준의 전문적인 문체로 재작성합니다.
  3. **목차 동기화 (Critical)**: 본문의 헤더(`##`, `###`) 내용이나 앵커가 변경된 경우, **반드시 문서 상단의 `## 목차` 섹션을 찾아 변경된 제목과 링크를 최신 상태로 업데이트**하여 문서 내 탐색이 깨지지 않도록 합니다.
  4. **가독성 최적화**: 인용구, 리스트, 코드 블록을 사용하여 구조를 잡습니다.
  5. **렌더링 방지 주석**: 원문은 반드시 다음과 같이 독립된 라인에 주석으로 감싸 렌더링 시 노출되지 않게 합니다.
     ```markdown
     
     <!-- ORIGINAL_START
     (사용자 원문 내용)
     ORIGINAL_END -->
     
     ```

## AI Expert Guidelines
- 사용자가 직접 고민한 흔적을 존중하되, 기술적인 엄밀함만 보완하세요.
- **일관성 유지**: 본문을 수정한 후에는 반드시 목차와 링크가 일치하는지 재검증하세요.
- 원문을 주석으로 처리할 때는 앞뒤에 빈 줄을 두어 마크다운 렌더러가 주석으로 정확히 인식하게 하세요.
- 만약 파일 내용이 부실하다면, 더 공부해서 내용을 채워달라는 정중한 요청과 함께 작업을 중단하세요.
